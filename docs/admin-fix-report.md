## 1. "활성 사용자" → "현재 접속자"로 변경

**태그:** 수정

### 증상
관리자 패널 현황 요약의 "활성 사용자" 항목이 실제 접속 여부와 무관하게 항상 고정된 숫자가 표시됨.

### 근본 원인
"활성"의 정의가 잘못되어 있었음 — `u.status === 'active'`는 계정이 비활성화되지 않았다는 뜻이지, 실제로 접속했다는 뜻이 아님. 접속 시각을 기록하는 필드(`lastActiveAt`) 자체가 존재하지 않았음.

### 이전 → 이후

| | 이전 | 이후 |
|---|---|---|
| **라벨** | 활성 사용자 | 현재 접속자 |
| **설명** | 정상 이용 중 | 최근 30분 내 접속 |
| **산정 기준** | `status !== 'disabled'`인 전체 유저 수 (항상 동일) | `lastActiveAt`가 30분 이내인 유저 수 (실시간) |

### 변경 내역

| 파일 | 위치 | 내용 |
|---|---|---|
| `index.html` | 3283-3285행 | `bootApp()`에서 `lastActiveAt` Firestore 기록 추가 |
| `js/admin.js` | 318-319행 | 30분 기준 `onlineNow` 필터 추가 |
| `js/admin.js` | 399-400행 | 라벨 "현재 접속자" + 카운트 `onlineNow.length`로 변경 |

## 2. 교회 목록이 표시되지 않는 문제 수정

**태그:** 수정

### 증상
현황 요약에서 "등록된 교회/기관 · 3개" 카드를 클릭하면 교회 목록 대신 *"교회를 찾을 수 없어요"* 빈 화면이 표시됨. 교회·기관 관리 섹션도 동일하게 비어있었음.

### 근본 원인
카드 클릭이 `openSubscreen('church-manage')`를 호출하는데, 이 화면은 **단일 교회 상세 화면**으로 `_viewingChurchCode`가 미리 설정되어야 동작함. 교회 코드 없이 열리면 `getChurchData(undefined)`가 null을 반환하여 빈 화면이 됨.

관리자 패널 페이지 하단에는 이미 교회 전체 목록이 렌더링되어 있었으나, 스크롤해야만 볼 수 있는 위치에 있어서 사용자가 발견하지 못한 것.

### 이전 → 이후

| | 이전 | 이후 |
|---|---|---|
| **카드 클릭** | `openSubscreen('church-manage')` | 같은 페이지 내 `#admin-church-section`으로 스크롤 |
| **결과** | `_viewingChurchCode` 미설정 → 빈 화면 | 교회 전체 목록으로 부드럽게 이동 |

### 변경 내역

| 파일 | 위치 | 내용 |
|---|---|---|
| `js/admin.js` | 377행 | onclick → `scrollIntoView({behavior:'smooth'})` |
| `js/admin.js` | 525행 | 교회 목록 섹션에 `id="admin-church-section"` 추가 |
